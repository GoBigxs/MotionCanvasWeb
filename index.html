Paper Code

MotionCanvas translates simple user movements into fine-styled artworks, such as flower paintings, allowing users to have engaging real-time art creation experiences. The canvas features a line color flowing effect to enhance appearance and reduce perceptive processing time. MotionCanvas also supports co-creation among multiple users within the same space. enhancing the interactive experience, forstering community environment, further encouraging the collaborative engagement in artistic creation.

Abstract

We introduce MotionCanvas, a spatial art creation tool that integrates Generative AI and motion capture technologies into artistic interactive experiences.

Existing interactive platforms provide engaging environments and allow users to interact through touch or gestures. However, they are often limited by predefined interactive content and gestures, lacking the ability to respond to user movements comprehensively.

We combine Generative AI models with motion capture technology to create novel forms of interactive art, with three primary objectives: 1) ensuring that the generated content aligns with the intended aesthetic, 2) advancing the possibilities for creative collaboration, and 3) minimizing latency within the interaction pipeline. The results of the user study confirm that our system can enhance user engagement and provides dynamic and immersive experiences.

System Workflow Overview

Initially, users engage with the system, initiating the sequence. Subsequently, the motion capture system records and transmits their movements, channeling raw data to MQTT topic one for processing. A Python program then undertakes the processing of data from topic one, subsequently disseminating refined information to MQTT topic two. The Unity program seamlessly integrates the processed data from topic two, facilitating the transformation of intricate 3D motion data into simplified 2D drawing inputs. These drawing inputs are subsequently forwarded to the Image Generation API, where a systematic server process generates elaborate flower images based on the provided data. Finally, the images are displayed on the screen.

Flower Generation Pipeline

The flower generation pipeline begins with the reception of point data from Unity, followed by a line processing that smooths lines and incorporates geometric shapes to enhance the base input image. Subsequently, the processed image undergoes conversion into a depth image using a Depth estimation model. Together with textual inputs, this depth-enhanced image is fed into the image generation module, crafted with selected components and fine-tuned parameters refined through iterative trials to optimize performance. The final step involves employing the BiRefNet background removal model to enhance visual clarity by eliminating background colors from the generated images.
